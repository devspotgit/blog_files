What is OAuth

OAuth (Open Authorization) is an open standard for access delegation. It allows user to grant limited access to a third-party application without sharing any credentials.

Why do we need OAuth

  • To avoid giving away passwords to third-party services
  • To limit what external apps can access (e.g., only profile info, not email or calendar)
  • To allow revoking access at any time
  • To support single sign-on (SSO) experiences like Login with Google

How OAuth Works

Below are the basic steps to login to an application through a Google account

  1. User clicks Login with Google on your app.
  2. The app redirects the user to Google’s OAuth authorization server.
  3. The user logs into Google and approves access.
  4. Google redirects back to your app with an authorization code.
  5. The app sends this code to Google’s server in exchange for an access token.
  6. The app uses this access token to access user data (e.g., email, name).

Types of OAuth Tokens

  • Authorization Code: Temporary code you exchange for a token
  • Access Token: Grants limited access to user resources
  • Refresh Token: Used to get a new access token without asking the user again

Common Use Cases

  • Social Login
  • 3rd-Party App Access
  • Delegated File Sharing
  • API Access Without Password

OAuth is the modern, secure way to authorize access between applications. It protects user data, improves user experience, and enables powerful integrations, all without ever sharing passwords.
